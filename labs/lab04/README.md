это абстрактный поток байтов, хранящийся на внешнем носителе и представляющий собой последовательность данных.

это указатель на структуру управления потоком, хранящую состояние файла, текущую позицию и настройки буферизации.

основные режимы: r (чтение), w (запись/перезапись), a (добавление) и их расширения + (для чтения+записи) и b (бинарный режим).

в текстовом файле данные могут преобразовываться (например, символы переноса строки), а в бинарном данные сохраняются как точная копия байтов из памяти.

функции для посимвольного ввода (чтения) и вывода (записи) в файл.

функции для работы со строками: fgets считывает строку с защитой от переполнения буфера, а fputs записывает строку целиком.

форматированный ввод и вывод (аналоги printf/scanf), позволяющие записывать в файл числа, строки и другие типы данных по шаблону.

функции для блочного чтения и записи "сырых" данных, обычно используемые для сохранения структур.

fseek устанавливает позицию "курсора" в файле, а ftell возвращает текущее расстояние от начала файла в байтах.

feof сообщает о достижении конца файла, а ferror фиксирует ошибки ввода-вывода (например, аппаратные сбои).

функция fclose необходима для корректного сохранения данных из буфера на диск и освобождения системных ресурсов.
